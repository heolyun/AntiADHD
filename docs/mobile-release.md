# Android beta release

AtiADHD Android releases are generated by `.github/workflows/mobile-release.yml`.
The workflow builds both an installable APK and a Play Store AAB. Release signing
credentials must never be committed to this repository.

## One-time signing setup

Create the permanent upload key on a trusted computer. Keep the resulting JKS and
its passwords in two separate backup locations. Losing this key can prevent future
APK updates from installing over the existing app.

```powershell
New-Item -ItemType Directory -Force mobile-app/release-signing
keytool -genkeypair -v `
  -keystore mobile-app/release-signing/antiadhd-release.jks `
  -alias antiadhd `
  -keyalg RSA `
  -keysize 4096 `
  -validity 10000
```

`keytool` asks for the passwords interactively. Do not paste those passwords into
source files or shell history.

Configure the repository variable used by the release build:

```powershell
gh variable set MOBILE_API_BASE_URL --body "http://172.30.1.39/api"
```

Configure these repository secrets in GitHub **Settings → Secrets and variables →
Actions**:

- `ANDROID_KEYSTORE_BASE64`: Base64 representation of the complete JKS file
- `ANDROID_KEYSTORE_PASSWORD`: JKS password
- `ANDROID_KEY_ALIAS`: `antiadhd`
- `ANDROID_KEY_PASSWORD`: key password

The JKS can be uploaded without printing its contents:

```powershell
$encoded = [Convert]::ToBase64String(
  [IO.File]::ReadAllBytes("$PWD/mobile-app/release-signing/antiadhd-release.jks")
)
$encoded | gh secret set ANDROID_KEYSTORE_BASE64
Remove-Variable encoded
```

Set the three short values interactively so they do not appear in command history:

```powershell
gh secret set ANDROID_KEYSTORE_PASSWORD
gh secret set ANDROID_KEY_ALIAS
gh secret set ANDROID_KEY_PASSWORD
```

## Build a beta

Open **Actions → Build Signed Android Release → Run workflow**, enter a semantic
version such as `0.2.0`, and run it. GitHub assigns an always-increasing Android
`versionCode` from the workflow run number. Download the APK artifact for direct
installation or the AAB artifact for Google Play internal testing.

Before distributing a build, verify login, schedule creation, AI planning, voice
recording, notification permission, and widget refresh on a physical device.

## Network boundary

The current API URL is a private LAN address and uses HTTP. This is acceptable only
for the trusted home network beta. Do not expose the k3s ingress directly to the
internet. External access must first use a VPN such as Tailscale, or a trusted domain
with HTTPS/TLS; then update `MOBILE_API_BASE_URL` and rebuild the app.
