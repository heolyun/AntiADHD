# AI 음성 일정 회귀 평가

사용 중 발견한 오인식 사례를 모델 학습 데이터처럼 취급하되, 실제 파인튜닝보다 먼저
프롬프트·응답 스키마·저장 전 확인 화면의 회귀 평가 사례로 관리한다.

| 입력 | 기대 동작 | 날짜 | 시간 | 비고 |
|---|---|---|---|---|
| 이번 주 금요일 친구랑 와인바 가기 | 일정 초안 | 이번 주 금요일로 해석 | 미정 | 시간 입력을 요청하고 Inbox로 보내지 않음 |
| 오늘 오후 7시에 운동하기 | 일정 초안 | 오늘 | 19:00 | 날짜와 시간이 모두 채워짐 |
| 매일 오후 7시에 운동하기 | 반복 루틴 초안 | 오늘부터 | 19:00 | DAILY |
| 언젠가 친구랑 와인바 알아보기 | 나중에 정리 | 미정 | 미정 | 일정 표현이 없으므로 Inbox |

새 실패 사례는 입력 문장을 그대로 추가하고, 기대 동작을 먼저 정의한 뒤 프롬프트와
파서를 수정한다. 운영 모델을 바꾸더라도 이 표의 동작이 유지되는지 확인한다.
